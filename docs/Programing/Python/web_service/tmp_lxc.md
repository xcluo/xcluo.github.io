## 智能评分系统

### 一、整体功能介绍

面向考核场景的智能化评分工具，通过语音识别（ASR）+ 智能评分算法，实现主观题回答的自动化解析与评分，提升评阅效率与客观性。

**核心流程**：批量导入考题 → 接收考生音频 → 语音转文本 → 智能分析评分 → 输出评分结果

| | 评分细则上传 | 上传评分规则；未上传时系统自动归纳 |
| **音频解析** | 音频接收/录制 | 上传文件或实时录制 |
| | ASR语音转写 | 音频转文本 |
| | 转写文本预览 | 可视化核对 |
| | 原始音频留存 | 支持复查存证 |
| **智能评分** | 要点智能解析 | NLP分析考生回答与预设要点 |
| | 要点匹配评分 | 自动比对打分 |
| | 实时结果输出 | 即时反馈评分及依据 |

## 智能评分系统 - 功能文档

### 一、整体功能介绍

**智能评分系统**是一款面向考核场景的智能化评分工具，通过整合语音识别（ASR）技术与智能评分算法，实现对考生主观题回答的自动化解析与评分，大幅提升主观题评阅效率与客观性。系统支持

- 考核方批量导入考题及评分标准；
- 考生音频回答内容转化为文本；
- 基于预设的答题要点与考生回答内容进行智能分析匹配，输出结构化评分结果，

---

### 二、核心功能模块

#### 模块一：考题管理
| 功能项 | 功能说明 |
|--------|----------|
| 批量导入 | 支持一次性上传多道主观题（至少包括题干内容、答题要点等内容），提升录入效率 |
| 评分细则扩展 | 可选上传详细评分标准（如分值权重、扣分规则等），若未上传该字段，智能评分系统将自动基于题干及答题要点信息进行归纳总结 |

---

#### 模块二：音频内容解析
| 功能项 | 功能说明 |
|--------|----------|
| 音频上传/录制 | 接收或录制考生提交的音频回答文件 |
| 语音转文本（ASR） | 自动将音频内容识别并转换为可编辑文本 |
| 原文保留 | 完整保存原始音频文件，支持回溯核查 |
| 文本预览 | 提供转写后的文字内容预览界面，便于快速核对 |

---

#### 模块三：智能评分引擎
| 功能项 | 功能说明 |
|---|---|
| 要点分析 | 将考生回答与预设答题要点进行智能分析比对 |
| 实时评分 | 基于分析结果即时计算并输出评分 |
| 评分反馈 | 提供评分依据说明，包括命中/遗漏要点 |

#### 硬件需求

为确保场地部署的稳定性，系统需采用负载均衡及数据层高可用架构，具体硬件配置如下：

- **负载均衡服务器**：1台，部署Nginx，负责将前端请求分发至业务服务器集群，实现流量调度与故障转移。
- **业务服务器**：2台，部署智能评分系统，构成业务处理集群，通过负载均衡共同承接请求，提升系统处理能力与可用性。
- **数据服务器**：3台，构成数据层服务集群，其中：
  - (主从架构)两台部署MySQL数据库与MinIO对象存储，提供结构化数据存储与文件存储服务；
  - 另一台部署Redis缓存，支撑高性能数据缓存及可能的哨兵或集群模式高可用。

**总计**：6台服务器。

### 工作难点
1. **登录依赖人工**：由于系统登录需手机验证码（暂未发现有静态登录方式），登录过程必须人工介入完成。
2. **信息提取准确要求高**：需迭代下载处理中工单（含申诉信息、附件、历史三个月关联工单），由于附件较多、篇幅较长，从复杂信息中准确提取指定信息难度较大，易造成错误传递，识别准确性要求较高。
3. **预处理和有/无责判别标准缺失**：尚无受理/不受理、有责/无责、退件的明确判别标准文档，且部分复杂工单必须人工介入，无法完全自动化。

### 计划安排
1. **初期自动化下载**：利用浏览器自动化工具，人工完成登录后，自动下载处理中的工单信息（含内容、附件、关联工单）。
2. **模块化处理开发**：基于工单处理工作流，开发文本、图片、视频、音频等专门处理模块，逐步提升自动化程度。
3. **后期逐步减少人工介入程度**：不断完善判别标准后，根据效果逐步降低人工介入比例，提升处理效率，释放人力。

### 资源需求
#### 标准需求（重要！）
由于暂无文档化判别标准，获取工单数据的后续操作均需依赖以下判别标准
- 工单受理/不受理的判别标准，为准确回执，需包括不受理的（10+个）子类判别标准
- 工单退回的判别标准
- 有责/无责判别标准

#### 硬件需求
主要为中端办公电脑配置，包括
- **系统**：windows或linux电脑（非kylin，防止playwright适配出问题）
- **CPU处理器**：由于需要对文本、文档的本地化处理（如进程并发，docx2image、pdf等），要求一定性能
- **GPU显卡**：用于布置可能需要专门的视觉或OCR模型
	> 同时可复用当前管局布置的大语言模型和多模态模型
- **内存条**：推荐DDR4 16g及以上
- **硬盘**：由于文件读写操作和存储文件需要较多，推荐512G及以上
